## B1 — Локация не создаётся в пустом аккаунте · `баг` · 🔴 блокер
**Симптом (со слов пользователя):** в новом пустом аккаунте **кнопки создания локации
вообще нет** — создать первую локацию невозможно (блокирует онбординг).

**Странность:** в коде кнопка «Добавить» в `LocationsPanel` (`entities/locations/list/locations-panel.tsx:47`)
рендерится безусловно, а вкладка «Локации» в `pages/finance/finance-page.tsx:45` — всегда.
Значит причина не в самой панели: либо страница Финансов в пустом аккаунте падает целиком
(белый экран → «кнопок нет»), либо вкладка/раздел недоступны новому пользователю, либо
точка входа фактически недостижима при онбординге.

**Метод:** superpowers: systematic-debugging. ВОСПРОИЗВЕСТИ на чистом аккаунте:
1. Зарегистрировать новый аккаунт (UI `/register` или API) — без локаций/групп/учеников.
2. Залогиниться, дойти до Финансы → вкладка «Локации». Зафиксировать, что именно видно
   (белый экран? нет вкладки? нет кнопки? консольная ошибка?).
3. Найти корень: проверить, не кидает ли какой-то хук на странице Финансов при пустых данных
   (`useFinancePage`, `usePayments`/`useHallCosts`/`usePricingRules`, дефолтная локация),
   не гейтится ли рендер вкладок/панели на наличие данных.

**Файлы (вероятные):** `pages/finance/finance-page.tsx`, `pages/finance/use-finance-page.ts`,
`entities/locations/list/locations-panel.tsx`, `entities/locations/api/use-locations.ts`,
`entities/finance/api/use-finance.ts`.

**Критерий приёмки:** на свежезарегистрированном аккаунте тренер может открыть раздел и
создать **первую** локацию; после создания она видна. `npm run build` чистый.

## B2 — Унификация выбора даты/времени · `баг`/рефактор · желательно
**Причина «воскресенья»:** `ConfigProvider` в `app/providers/theme-provider.tsx:44` НЕ
передаёт `locale`, поэтому antd-пикеры используют дефолт (англ., неделя с воскресенья).
`dayjs.locale('ru')` уже выставлен в `common/utils/date.ts`, но antd берёт первый день
недели из своей локали.

**Решение:**
1. В `theme-provider.tsx` импортировать `ruRU from 'antd/locale/ru_RU'` и передать в
   `<ConfigProvider locale={ruRU} theme={...}>`. Это переключает все DatePicker/TimePicker/
   Calendar на ПН-первый день и русские подписи разом.
2. Унифицировать формат отображения даты: везде, где `<DatePicker>` без `format`, задать
   `format="DD.MM.YYYY"` (эталон — `mark-paid-modal.tsx:188`). Время — `format="HH:mm"`
   (уже так в локациях). Найти все вхождения: `grep -rl "DatePicker\|TimePicker" apps/frontend/src`.
3. (Опционально, если просто) — тонкая обёртка `common/ui` над antd DatePicker с дефолтным
   форматом, чтобы не дублировать. Не обязательно, если правок немного.

**Файлы:** `app/providers/theme-provider.tsx` (главное), плюс пройтись по DatePicker'ам:
`entities/finance/form/*`, `entities/students/form/student-form-modal.tsx`,
`entities/students/subscriptions/renew-sub-modal.tsx`, `entities/trainings/form/*`.

**Критерий:** во всех календарях/пикерах (создание абонемента, тренировки, фильтры) неделя
начинается с **понедельника**, формат дат единый `DD.MM.YYYY`. `npm run build` чистый.

## B4 — Навигация по периодам в статистике · `фича` · желательно
**Решение:** стрелки ← → для листания периодов + сохранить пресеты Месяц/Квартал/Год/Всё +
подпись текущего периода.

**Что есть:** `entities/finance/stats/use-finance-stats.ts` — `FinancePeriod`
(`month/quarter/year/all`), `periodStart()` считает начало от **текущего** момента (`new Date()`),
нет смещения. `stats-tab.tsx` рисует кнопки пресетов и зовёт `stats.setPeriod`.

**Шаги:**
1. В `use-finance-stats.ts` добавить состояние `offset` (число, 0 = текущий период).
   Сделать `periodRange(period, offset): { start, end }` — границы периода со сдвигом:
   - month: первое/последнее число месяца `now.month - offset`;
   - quarter: границы квартала со сдвигом на `offset` кварталов;
   - year: границы года `now.year - offset` (или +/−);
   - all: без границ, стрелки скрыть.
   Фильтровать `payments` по `paid_at >= start && paid_at <= end` (а не только `>= start`).
2. Экспортировать из хука: `offset`, `prev()`, `next()` (next не уходит в будущее дальше 0
   — текущий период максимум), и `periodLabel` (напр. «Июнь 2026» / «Q2 2026» / «2026»;
   для `all` — пусто). Использовать `dayjs`/`toLocaleString('ru', …)`.
3. В `stats-tab.tsx` добавить ряд навигации: `‹` + `periodLabel` + `›` (скрыть для `all`,
   `›` задизейблить на текущем периоде). Стиль — рядом с `.fin-period-selector`.

**Файлы:** `entities/finance/stats/use-finance-stats.ts`, `entities/finance/stats/stats-tab.tsx`
(+ `.scss` при необходимости).

**Критерий:** можно пролистать на прошлый месяц/квартал/год и увидеть исторические суммы;
подпись периода корректна; вперёд за текущий период не уходит. `npm run build` чистый.
