# Редизайн главной: агенда, блок внимания, мобильные действия, KPI

**Дата:** 2026-06-11 · **Ветка:** `feat/home-redesign`
**Источник:** `план/trikick-home-redesign-prompt.md` (промпт по дизайну пользователя) + решения, зафиксированные в обсуждении.

## Решения продукта (зафиксированы пользователем)

1. **«Не оплачено»** — бейдж только у **прошедших** занятий, за которые оплата не
   записана. Будущие занятия тревожных плашек не получают.
2. Кнопка «···» с десктоп-макета — декор, **не реализуем**.
3. Планшетный режим со сворачиванием сайдбара — **выкинут** (YAGNI, трогает общий layout).

## Что уже сделано до этой фичи (вне объёма)

Нижний таб-бар (общий компонент), KPI 4-в-ряд/2×2 (брейкпоинт 1100px), короткие
подписи кнопок ≤480px, перенос действий строкой под заголовком ≤768px, две
колонки 1.5fr/1fr на ≥1024px (`home-cols`).

## Ключевая находка разведки

`GET /api/students` **уже отдаёт `billing`** у каждого визита
(`'subscription' | 'payment' | 'none'`) — Sequelize сериализует всю модель Visit.
Фронт это поле отбрасывает при маппинге. Значит, точный статус оплаты занятия
доступен **без единого изменения API/бэкенда** — только пробросить поле через
`RawVisit` → `VisitRecord` в `students.repo.ts` / `types.ts`.

## 1. Агенда вместо таймлайна

**Новый компонент** `pages/home/today-agenda.tsx` (+`.scss`). `TrainingDayCalendar`
используется только главной — после замены удаляется отдельным коммитом.

**Данные:** `home-page.tsx` мемоизирует `buildCalendarDay(todayISO(), trainings,
students, groups).day.blocks` (модель `CalendarBlock` уже даёт `time`, `label`,
`isInd`, `isScheduled`, `attendeesCount`, `trainingId`, `groupId`) и передаёт в
агенду пропом `rows`. `calendar-model.ts` не меняется. Клик по строке — прежний
контракт `onBlockClick(block)` → `CalendarTrainingModal`.

**Строка:** `время · длительность | кружок (группа — иконка TeamOutlined в
--accent-dim; инди/пара — инициалы в --ind-dim) | label | подтип | статус`.
Подтип: `trainings.type.*` по `training.isPair / isOnline / group.isIndividual`;
для группы — ещё «N чел.» при `attendeesCount > 0`. Длительность строки:
`training.sessionDuration || group.duration || 60`.

**Время:** `now` в state, обновляется интервалом 30 с. Прошедшее = `start + dur <= now`.

**Статусы справа:**
- прошедшее, посещения отмечены, все визиты `billing !== 'none'` → зелёная галочка;
- прошедшее, отмечены, есть визит `billing === 'none'` → янтарная плашка «Не оплачено»;
- прошедшее, посещений нет (включая scheduled-слоты) → приглушённое «Не отмечено»;
- идёт сейчас (`start <= now < end`) → индиго «Идёт» + рамка;
- ближайшее будущее → индиго «Скоро» + рамка 1.5px `--accent`; остальные будущие — без бейджа.

Визиты ищутся по `trainingId` в `student.visitHistory` (map `trainingId → billings[]`
в `useMemo` по `students`). Прошедшие строки — `opacity: 0.55`.

**Разделитель «СЕЙЧАС · HH:MM»:** между прошедшими и остальными, тонкие линии
`--accent` по бокам; показывается всегда при непустом списке.

**Пустое состояние:** `EmptyState` (иконка `CalendarOutlined`, текст
`home.noTrainingsToday`, кнопка `home.recordTraining` → открытие выбора типа).

## 2. Блок «Требуют внимания» (объединение)

В `home-page.tsx` собирается единый список карточек на базе `WarningItem`
(компонент не меняется — полоса 3px уже есть: дефолт `--warning`, `--danger` через проп):

| Источник | Полоса | detail | Действие |
|---|---|---|---|
| `warnings`, `status.type==='expired'` | красная | как сейчас | «Продлить» (primary) → `RenewSubModal` |
| `unpaid` из `useUnpaidSubs` | янтарная | группа/инд · «Ожидает оплаты» | «Записать оплату» (outline) → `MarkPaidModal` |
| `warnings`, `status.type==='ending'` | янтарная | как сейчас | «Продлить» (primary) |

Порядок: expired → unpaid → ending. Клик по карточке — `StudentDrawer` (для всех).
Один абонемент может породить две карточки (истёк И не оплачен) — это две разные
проблемы с разными действиями, не дедупим. Заголовок «Требуют внимания» при
непустом списке (учитывая unpaid), иначе «Статус абонементов» + ok-карточка.
Кнопка-пилюля «Ожидают оплаты» в шапке остаётся (статус-сигнал + модалка).

## 3. Мобильная строка действий (<768px)

`PageHeader` не меняется — оборачивается в `<div className="home-header">`,
стили таргетят `.home-header .page-header`:
- шапка остаётся строкой (переопределяем column из `responsive.scss`): заголовок
  слева, справа только «+» (иконка, текст скрыт); кнопки markToday/unpaid в шапке скрыты;
- под шапкой блок `.home-actions` (виден только <768px): зелёная пилюля
  «Отметить за сегодня» (`flex: 1.4`) и янтарная «Ожидают оплаты (N)» (`flex: 1`),
  высота ≥44px. Янтарная: фон `--warning-dim`, текст/иконка `--warning`.
  Обработчики те же, что у кнопок шапки.

На десктопе всё как сейчас (три кнопки в шапке справа).

**Подзаголовок даты:** `PageHeader subtitle` = «Четверг, 11 июня · 3 тренировки»
(`dayjs().format('dddd, D MMMM')` с капитализацией + плюрал-ключ по числу строк агенды).

## 4. Приглушение нулевых KPI

`KpiCard` получает опц. проп `dimZero?: boolean`; при `dimZero && value === 0`
класс `kpi-card--dim`: иконка на `--surface-2`/`--text-muted`, значение
`--text-muted`. Передаётся только с главной (другие экраны не затронуты).

## Новые i18n-ключи (ru / en)

- `home.agendaNow`: «СЕЙЧАС» / "NOW"
- `home.agendaSoon`: «Скоро» / "Soon"
- `home.agendaInProgress`: «Идёт» / "Now on"
- `home.agendaUnpaid`: «Не оплачено» / "Unpaid"
- `home.agendaUnmarked`: «Не отмечено» / "Not marked"
- `home.agendaAttendees`: «{{count}} чел.» / "{{count}} ppl."
- `home.trainingsCount` (+`_few`/`_many` ru; `_one`/`_other` en): «{{count}} тренировка…» / "{{count}} training(s)"
- `home.attentionUnpaid`: «Ожидает оплаты» / "Awaiting payment"
- `home.minutesShort`: «{{count}} мин» / "{{count}} min"

Кнопка оплаты — существующий `students.subCard.markPaid` («Записать оплату» —
единый термин проекта, НЕ «Отметить оплату» из промпта).

## Ограничения

- Палитра — только токены проекта (`--accent #5856D6`, `--success`, `--warning`,
  `--danger`, dim-версии); хексы макета не используются.
- Другие экраны и общие компоненты не трогаем (исключения: opt-in проп `KpiCard`,
  проброс уже отдаваемого поля `billing` в маппинге учеников).
- Слой запросов/API не меняется.
