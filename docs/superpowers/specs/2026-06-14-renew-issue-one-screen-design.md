# Продление/оформление абонемента «в один экран» — дизайн

Дата: 2026-06-14 · Ветка: `feat/my-day-quick-actions`

## 1. Цель

Привести модалку `RenewSubModal` (продление и оформление абонемента) к утверждённому
макету «3а · Продление в один экран» из брейншторма
(`.superpowers/brainstorm/22432-1781301831/content/day-design-detail.html`).

Сейчас модалка — обычная вертикальная antd-форма (выпадашка «Тип абонемента»,
календарь «Дата начала», свитч). Нужен компактный одноэкранный вид со строками-плашками
«лейбл слева · значение справа», как в макете.

Функциональность (префилл из прошлого абонемента, цена из тарифа, «оплачен сразу» →
авто-платёж) **уже реализована** и сохраняется без изменений — меняется только подача
и убираются лишние поля.

## 2. Что на экране (режим «Продлить»)

```
Продлить: Маша · «Старт»
┌──────────────────────────────────────┐
│ Как прошлый        8 занятий · 9 600 ₽ │
├──────────────────────────────────────┤
│ Начало             сегодня ▾          │
└──────────────────────────────────────┘
☑ Оплачен сразу
  создастся платёж 9 600 ₽
            [ Продлить ]
```

- **Заголовок:** `Продлить: <Имя> · «<Группа>»` (имя ученика и группа прямо в заголовке;
  отдельной строки «Ученик/Группа» больше нет).
- **Как прошлый** → `<N> занятий · <цена> ₽` — справочная плашка: что именно будет создано.
  Тип абонемента (число занятий) и цена берутся из последнего абонемента ученика в этой
  группе + тарифа. **Не редактируется** (см. §4 — решение об убранном выборе типа).
- **Начало** → быстрый выбор `Сегодня · Завтра · Выбрать дату…`. По умолчанию «Сегодня».
  «Выбрать дату…» открывает обычный календарь.
- **Оплачен сразу** — строка с галочкой/свитчем и подписью `создастся платёж <цена> ₽`
  (подпись видна только когда включено и цена известна). По умолчанию включено.
- Кнопка **Продлить** (primary, на всю ширину футера).

## 3. Режим «Оформить» (issueMode)

Тот же компактный вид, но:
- Заголовок: `Оформить: <Имя> · «<Группа>»`.
- Прошлого абонемента нет → плашка называется не «Как прошлый», а нейтрально
  (`Абонемент` / `Тариф`), показывает дефолт **разовое занятие** + цену.
- Кнопка: **Оформить**.

## 4. Убранные поля (решение)

- **Тип тренировки** (групповой/индивидуальный/онлайн/парная) — НЕ показываем и НЕ выбираем.
  Он однозначно задан контекстом: продление/оформление всегда происходит на конкретной
  группе или индивидуальном занятии. `lessonKind` выводится автоматически из группы
  (`group.isIndividual`), как и сейчас в коде.
- **Тип абонемента** (1/4/8 занятий) — выбор убираем. Продление идёт «как прошлый»:
  число занятий копируется из последнего абонемента. Плашка «Как прошлый» — справочная.

> **Решение по issue-mode дефолту (утверждено).** В режиме «Оформить» прошлого
> абонемента нет, копировать нечего. Дефолт — **разовое занятие** (`type = '1'`).
> То есть оформление без прошлого абонемента создаёт разовое посещение по тарифу
> группы (`single`-формат). Меняем текущее поведение кода `prevSub?.type ?? '8'`
> на `prevSub?.type ?? '1'`.

## 5. Поведение

- Цена и подпись «создастся платёж …» — живые: считаются `resolvePricingRule` по кортежу
  `subTypeToTuple(type, group.isIndividual)` (как сейчас). Так как тип теперь фиксирован,
  пересчёт происходит только при открытии (тариф/слот не меняются внутри окна).
- Слот прайм/обычный наследуется от прошлого абонемента (`prevSub?.timeSlot`) — без изменений.
- «Оплачен сразу» ON → создаётся абонемент + авто-платёж, связывается с абонементом
  (`linkPaymentToSub`); OFF → только абонемент, попадает в «Ожидают оплаты». Логика та же.
- «Начало»: значение — ISO-дата. «Сегодня» = `todayISO()`, «Завтра» = `todayISO()+1`
  (нужен новый хелпер `tomorrowISO()` в `common/utils/date.ts` по образцу `yesterdayISO`),
  «Выбрать дату…» — antd DatePicker со значением выбранной даты.

## 7. Компоненты выбора даты

Быстрый выбор «Начало» — небольшой внутренний под-компонент (или Segmented/Select antd
с тремя пунктами). При выборе «Выбрать дату…» под строкой раскрывается DatePicker.
Состояние: `dateMode: 'today' | 'tomorrow' | 'custom'` + `customDate`. Итоговая `date`
вычисляется чистой функцией — её и покрываем тестом.

## 8. Тесты (gate)

- **Jest (чистые функции):** `tomorrowISO`; функция вычисления итоговой даты из режима.
- **Сборка фронта:** `npm run build -w apps/frontend` (tsc+vite) — 0 ошибок.
- **eslint:** 0/0.
- **Ручная проверка (Playwright/браузер):**
  - «Продлить» у Маши (Старт · Заканчивается): заголовок «Продлить: Маша · «Старт»»,
    плашка «8 занятий · цена», «Начало: сегодня», свитч ON с подписью; «Продлить»
    создаёт абонемент + платёж.
  - «Оформить» у Пети/Ольги (нет абонемента): заголовок «Оформить: …», дефолт разовое занятие.
  - «Начало → Завтра» и «→ Выбрать дату» меняют дату создаваемого абонемента.
