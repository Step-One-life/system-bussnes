# «Мой день»: быстрая отметка и деньги в один экран — дизайн

Дата: 2026-06-13. Статус: утверждён StepOne (брейншторм с мокапами, вариант A «Эволюция ленты»).

## 1. Контекст и цель

Главная уже наполовину «пульт дня»: лента занятий с разделителем «Сейчас», KPI-полоса,
колонка «Требует внимания», кнопки «Отметить день» и «Оплаты». Боли пользователя
(основное устройство — телефон, 80%+ операций):

1. **Отметка посещений** — много шагов; нужны два потока: точечная отметка сразу после
   занятия (1–2 тапа) и пакетное закрытие дня вечером.
2. **Денежные операции** — продление абонемента и «отметить оплату» требуют длинных форм.

Цель: 90% дневного цикла тренера выполняется с главной без переходов, mobile-first.
Выбранный подход — эволюция текущей главной (без слома недавнего редизайна), а не
карточка «Сейчас» или inbox-очередь задач (рассмотрены и отклонены: каждая жертвует
одним из двух потоков отметки).

## 2. UX-механики

### 2.1 Быстрая отметка из ленты дня

- В строках ленты у прошедших и идущих занятий без отметки — кнопка **«Отметить»**
  (тап по самой строке по-прежнему открывает общую модалку занятия).
- Кнопка открывает шторку: участники **предотмечены** («все пришли») — состав группы
  для групповых, плановые ученики для индивидуальных/парных. Пользователь снимает
  только отсутствующих.
- Под каждым именем — превью биллинга: «спишется занятие · осталось 3» /
  «разовый платёж · 1 200 ₽» / «нет тарифа».
- Кнопка подтверждения: **«Готово · N из M»**.

### 2.2 «Закрыть день» (пакетно)

- Эволюция модалки «Отметить день»: один экран со всеми занятиями дня.
- У каждого занятия одна галочка — «прошло как запланировано» (все предотмеченные
  пришли). Уже отмеченные показаны приглушённо с ✓.
- Тап по занятию раскрывает тот же список участников, что в 2.1 (точечная правка).
- Кнопка: **«Отметить всё · N занятий, K человек»**.
- Работает с любой датой (сегодня, вчера).

### 2.3 Деньги в один экран

- **Продление**: шторка с предзаполнением из последнего абонемента ученика в группе
  (тип, число занятий) и ценой из тарифа; переключатель «оплачен сразу» (создаёт
  платёж); кнопка «Продлить». Поля редактируемы, но в типовом случае не трогаются.
- **Отметить оплату**: шторка с суммой из абонемента/тарифа и датой «сегодня»;
  кнопка «Оплачен ✓».
- Доступны из всех текущих точек: главная («Требует внимания», модалка оплат),
  карточка/дровер ученика.

### 2.4 Новый сигнал «Требует внимания»

- «Вчера не отмечено N занятий» → открывает «Закрыть день» за вчера.
- Серьёзность в колонке (по убыванию): истёкшие абонементы → неотмеченный вчерашний
  день → неоплаченные → заканчивающиеся.

### 2.5 Мобильная форма

- Все новые поверхности — `adaptive-sheet`: на мобиле antd Drawer `placement="bottom"`
  (грабер, крупные зоны тапа), на десктопе — Modal. Один компонент, два режима по
  брейкпоинту.
- Стили — только токены `--tk-*` (шторка `--tk-surface-raised`, поля
  `--tk-surface-card`, радиусы `--tk-radius-card/control`, кнопки `tk-btn-primary`/
  `tk-btn-secondary`, сигнал «не отмечено» — `--tk-warning-subtle-*` с янтарной
  точкой). Правило subtle-accent сохраняется: индиго — только мелким элементам.
- Мокапы брейншторма: `.superpowers/brainstorm/22432-1781301831/content/`
  (`day-design-tokens-v2.html`) — локальная папка, в git не входит.

## 3. Данные и API

**Бэкенд не меняется.** Все операции на существующих эндпоинтах:

- `createTraining` — для слота расписания без созданной тренировки;
- `markAttendance` (`POST /api/trainings/:id/attendees`) → `{ training, billing[] }` —
  идемпотентен, транзакции с `FOR UPDATE` уже на сервере;
- `removeAttendee` — снятие отметки с точным откатом биллинга (`revertVisit`).

Фронтовая логика:

- `use-mark-today` обобщается в **`useDayMarking(date)`**: построение списков
  «отмеченные ∪ плановые» и циклы сохранения берутся как есть, дата параметризуется.
- **Изменение дефолта**: для неотмеченных занятий галочки по умолчанию стоят у всех
  ожидаемых участников (сейчас — только у уже отмеченных). Для уже отмеченных занятий
  дефолт прежний (фактические отметки).
- Превью биллинга на клиенте: `findSubForGroup` (зеркало серверного
  `pick-subscription`) + `pricing-lookup`. Это оценка; источник истины — `billing[]`
  из ответа сервера, показывается в итоговом тосте.
- Prefill продления: последний по дате создания абонемент ученика в этой группе;
  цена — из `pricing-rules`. Prefill оплаты: сумма абонемента, дата — сегодня.

## 4. Компоненты

| Компонент | Где | Суть |
|---|---|---|
| `AdaptiveSheet` | `common/ui/adaptive-sheet` | Drawer (bottom, мобайл) / Modal (десктоп), один API: title, footer, open/onClose. |
| `QuickMarkSheet` | `entities/trainings/quick-mark-sheet` | Участники одного занятия: предотметка, биллинг-превью, «Готово · N из M». Открывается из ленты и из «Закрыть день». |
| `agenda-model` | `pages/home` | Флаг `needsMark` (прошедшее/идущее без отметки) → кнопка «Отметить» в строке. |
| `CloseDaySheet` | `pages/home` (бывш. `mark-today-modal`) | На `useDayMarking(date)`: галочка на занятие, «Отметить всё», раскрытие в `QuickMarkSheet`. |
| `RenewSubModal` / `MarkPaidModal` | `entities/students` / `entities/finance` | Компактный предзаполненный режим; prefill-хелперы — чистые функции (`renew-prefill.ts` и т.п.). |
| Сигнал «вчера» | `pages/home` | Элемент в «Требует внимания» при наличии неотмеченных занятий за вчера. |

## 5. Ошибки

- Сбой посреди пакетного сохранения: текущий паттерн сохраняется — тост с ошибкой +
  инвалидация кэшей (`trainings`/`students`/`groups`) в `finally`; повторное
  сохранение безопасно (идемпотентность сервера).
- «Нет тарифа» — существующее предупреждение `finance.autoRecord.noTariff` остаётся
  (имена учеников без тарифа в тосте).
- Расхождение превью и факта решается в пользу сервера: тост показывает фактический
  результат биллинга.
