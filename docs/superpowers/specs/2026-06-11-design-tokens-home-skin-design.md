# Дизайн-токены (2 темы, system) + рескин главной

**Дата:** 2026-06-11 · **Ветка:** `feat/design-tokens`
**Источник:** `план/trikick-tokens-and-home-redesign-prompt.md` (словарь токенов и
палитра — там, здесь только инженерные решения и дельты).

## Решения

1. **Префикс `--tk-`.** Имена словаря (`text-primary`, `border-focus`…)
   коллидируют со старыми токенами `tokens.scss`, на которых живёт всё
   приложение. Новые семантические токены — `--tk-*` в новом файле
   `app/theme/design-tokens.scss` (обе темы под `[data-theme=…]`,
   README-комментарий в шапке). Старый `tokens.scss` не трогаем, кроме:
   значения `--bg` приводятся к `surface-page` (#0a0a0c / #faf9f6) — фон
   страницы общий для layout, расхождение фона между экранами хуже, чем
   едва заметный сдвиг тона на старых экранах.
2. **Тема.** `ThemeProvider`: `preference: 'light' | 'dark' | 'system'`
   (localStorage `tk_theme`, легаси-значения light/dark остаются валидными,
   дефолт без записи — `system`), resolved `mode` — из preference или
   `matchMedia('(prefers-color-scheme: dark)')` с подпиской на изменения.
   `data-theme` всегда получает resolved-значение — обе системы токенов
   работают без правок. Меню профиля: вместо одного пункта-переключателя —
   три пункта с галочкой (Светлая/Тёмная/Системная).
3. **antd не перекрашиваем.** `colorPrimary` и компонентные токены antd
   остаются старыми (другие экраны). Нейтральные кнопки главной — классы
   `tk-btn-primary` / `tk-btn-secondary` поверх antd Button (как btn-mark).
4. **Роль индиго на главной** — только состояние: подсветка текущего/
   ближайшего занятия, разделитель «Сейчас», активный пункт меню.
   Кнопки: «+ Занятие» — `tk-btn-primary` (светлая лаванда #e8e6f8 с тёмным
   текстом в тёмной теме; графит в светлой); «Отметить день» и «Ожидают
   оплаты» на десктопе — `tk-btn-secondary` (нейтральная с обводкой), у
   оплат — янтарный бейдж-счётчик; мобильные пилюли — success-subtle и
   warning-subtle заливки. «Продлить» — `tk-btn-primary`, «Записать
   оплату» — `tk-btn-secondary`.
5. **WarningItem**: цветная точка статуса вместо левой полосы 3px, новые
   токены. Компонент используется только главной (home-page,
   kpi-detail-modal) — другие экраны не задевает.
6. **Агенда**: «не оплачено» — warning-текст с маленькой точкой; прошедшие
   opacity 0.5; время/счётчики — `tabular-nums` (токен-классом).
7. **Вне охвата**: остальные экраны, antd-модалки главной (mark-today и
   проч. мигрируют вместе со всеми), favicon.

## Definition of done

- `design-tokens.scss` с обеими темами и комментарием-инструкцией.
- system-режим работает: смена системной темы на лету меняет приложение,
  ручной выбор переживает перезагрузку.
- Главная (страница + её компоненты) — без сырых хексов, только `--tk-*`
  (старые `--sp-*`/`--dur`/`--ease` геометрия допустима).
- Gate: сборки фронт/бэк, eslint, jest. Визуальная приёмка — 4 скриншота
  пользователя (desktop/mobile × dark/light), Playwright не используется
  по его просьбе.
