# Видимость плановых учеников при отметке + фильтр «Ожидают оплаты» — дизайн

Дата: 2026-06-10. Ветка: `fix/planned-attendance-visibility`.
Пункты аудита: №5, №6, №7. Только фронтенд — биллинг отметки уже целиком на бэке
(`POST /trainings/:id/attendees`, `DELETE /trainings/:id/attendees/:studentId`,
см. спеку `2026-06-10-unified-attendance-billing-design.md`).

## Проблемы

1. **№5, «Отметить сегодня»** (`pages/home/use-mark-today.ts`): список строк
   индивидуальных занятий строится тремя ветками: «есть attendees → показать только
   их», «парное плановое», «плановое серии». Как только у парного занятия отмечен
   один ученик, срабатывает первая ветка и второй (плановый) пропадает из списка —
   занятие, отмеченное наполовину, нельзя доотметить.
2. **№5-бис, скрытый баг там же:** `indChecks` ключуется по `trainingId`, а у
   парного занятия две строки с одним `trainingId` — галочки обоих учеников
   сцеплены: отметить только одного из пары из этой модалки невозможно.
3. **№6, календарная модалка** (`entities/trainings/form/use-calendar-training.ts` +
   `calendar-training-modal.tsx`): для индивидуальных/парных занятий `indRows`
   строится только из `training.attendees` — строки с крестиком удаления, без
   чекбоксов. Плановые ученики (`plannedStudentId`, `plannedStudentId2`) не видны
   вовсе → плановое занятие нельзя отметить из календаря, то есть нельзя отметить
   задним числом (только через «Отметить сегодня» и только сегодняшние).
4. **№7, виджет «Ожидают оплаты»** (`pages/home/use-unpaid-subs.ts`): показывает
   все абонементы без `finPaymentId` навсегда, включая истёкшие год назад —
   вечный шум.

## Утверждённые продуктовые решения

- Календарная модалка индивидуальных/парных: **чекбоксы, как у групповых** —
  единый вид во всём приложении. Отмеченные с галочкой, плановые без; снятие
  галочки = снятие отметки с откатом денег (бэк), установка = отметка со
  списанием/платежом.
- Даты отметки из календаря: **без ограничений** (прошлое, сегодня, будущее) —
  как сейчас у групповых; платёж в любом случае записывается датой занятия
  (`paid_at = training.date`).
- «Ожидают оплаты»: показывать **действующие + истёкшие не более 30 дней назад**.

## 1. Календарная модалка (№6)

### `use-calendar-training.ts`

- `indRows` строится из объединения с дедупликацией:
  `attendees ∪ plannedStudentId ∪ plannedStudentId2` (null-ы отбрасываются).
  Каждая строка — существующий `AttendRow` (имя, subLine, статус абонемента).
- `checked` инициализируется из `attendees` (как сейчас) — плановые без галочки.
- `saveAttendance` обобщается: `toAdd = checked − attendeeSet`,
  `toRemove = attendeeSet − checked`. Сейчас `toRemove` считается от `groupRows`
  (ученики группы), что для индивидуальных не работает; формула от `attendeeSet`
  корректна для обоих типов (в `attendeeSet` только реально отмеченные).
- Функция `removeAttendee` (точечное удаление с тостом) больше не нужна модалке —
  удаляется из возвращаемого API хука.

### `calendar-training-modal.tsx`

- Ветка `m.isInd` рендерит `indRows` через тот же `RowItem`, что групповые:
  `checkbox`, `checked={m.checked.has(...)}`, `onToggle`; крестик удаления
  (`onRemove`) убирается.
- Футер для индивидуальных: добавляется кнопка «Сохранить» (`m.saveAttendance`,
  как у групповых), кнопка «Добавить» (ученика) остаётся.
- Пустой список (нет ни отмеченных, ни плановых — легаси-занятия) — текущая
  заглушка `trainings.cal.noStudents` остаётся.

## 2. «Отметить сегодня» (№5)

### `use-mark-today.ts`

- Три ветки построения строк схлопываются в одну: для каждого индивидуального
  занятия `ids = attendees ∪ plannedStudentId ∪ plannedStudentId2` (дедуп),
  строка `{ trainingId, studentId, time, groupId, originalPresent: attendees.includes(sid) }`.
- Ключ чекбоксов меняется с `trainingId` на составной **`` `${trainingId}:${studentId}` ``**:
  `indChecks`, `initChecks`, `toggleInd(trainingId, studentId)`, чтение в `save`.
- `save` для индивидуальных не меняется по сути: добавление через
  `markAttendance(tr, [studentId])`, снятие через `removeAttendee` — уже
  посимвольно per-student, нужно лишь читать галочку по новому ключу.

### `pages/home/mark-today-modal.tsx` (рендер `todayIndividuals`)

- Ключ строки и обработчик переключения переводятся на пару
  (`trainingId`, `studentId`).

## 3. «Ожидают оплаты» (№7)

### `use-unpaid-subs.ts`

К условию `!sub.finPaymentId` добавляется фильтр срока через существующий
`getDaysRemaining(sub)` (`entities/students/model/subscription-status`):

- `days === null` (нет `expiresAt`) → показывать;
- `days >= -30` → показывать (действующие и истёкшие ≤30 дней назад);
- иначе скрыть.

Абонемент с `remaining = 0`, но не истёкшим сроком, остаётся в списке — ученик
отходил неоплаченный абонемент, это актуальный долг. Порог 30 — константа
`UNPAID_EXPIRED_GRACE_DAYS = 30` в этом же файле.

## 4. Чего НЕ делаем

- Бэкенд не трогаем (эндпоинты и биллинг готовы).
- Пункт №9 (subLine не учитывает длительность 60/90 и общие абонементы) — отдельная
  задача, отображение не меняем.
- Удаление планового ученика «насовсем» из занятия — это редактирование занятия,
  не отметка; в модалке не появляется.
