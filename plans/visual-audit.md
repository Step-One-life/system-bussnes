# План визуальной сверки фронта с legacy

Цель — пройти каждый экран и элемент, сравнить с `legacy/` и устранить
расхождения в оформлении: размеры кнопок, типографика, отступы, адаптив.

**Эталон:** `legacy/style.css`, `legacy/index.html`, `legacy/*.js` (render-функции).
**Метод:** для каждого пункта — открыть страницу в браузере (desktop 1280
и mobile 390), сравнить с legacy-классом, зафиксировать расхождение, починить.

## Корневые причины (выявлены предварительно)

1. **Кнопки.** Фронт использует antd `<Button>` с дефолтными размерами antd
   (`padding`, `height`, `font-size` отличаются от legacy `.btn`). Legacy:
   - `.btn` — `padding: 10px 20px`, `font-size: 0.875rem`, `font-weight: 600`
   - `.btn--sm` — `padding: 4px 12px`, `font-size: 0.8rem`
   - `.btn--lg` — `padding: 12px 24px`, `font-size: 0.95rem`
   antd-кнопки крупнее/иначе → ломают раскладку.
2. **Нет адаптивных брейкпоинтов.** Legacy имеет `@media (max-width:480px)` и
   `(max-width:680px)` — уменьшает `page-title`, кнопки в хедере, KPI и т.д.
   Фронт этих правил не перенёс → текст наезжает.
3. **PageHeader.** При `flex-wrap` + крупные antd-кнопки → кнопки наезжают на
   заголовок на узких экранах. Legacy: `.page-header .btn` на мобиле мельче.
4. **Размеры шрифтов/отступов** местами не сверены поэлементно.

## Стратегия решения (УТВЕРЖДЕНО)

- **Кнопки:** конфиг antd `theme.components.Button` в `antd-theme.ts`
  (`controlHeight`, `controlHeightSM`, `controlHeightLG`, `paddingInline`,
  `fontSize`, `borderRadius`) под размеры legacy `.btn`. Компоненты не трогаем.
- **Адаптив:** перенести брейкпоинты legacy 1:1 — `480px`, `680px`, `768px`.
  Создать общий `app/theme/responsive.scss` с media-правилами (page-title,
  page-header кнопки, KPI-сетка, календарь, finance, модалки).
- Прочие точечные расхождения — править в `.scss` соответствующего блока.

   закрывает большинство «наездов» разом.
2. Затем layout (п.1-2).
3. Постранично 3-8, каждую проверять desktop+mobile в браузере.
